Decisions baked into this design

  • Reviewed = explicit numeric value (incl. 0) OR explicit "No change" click. NULL = unreviewed. Existing schema fields increase_pct + increase_dollar support this if we stop defaulting to 0 at line creation.
  • "No change" button writes increase_pct = 0 explicitly with a review_action="no_change" log entry to budget_revisions. Distinct from a typed 0 so the FA can see which lines got individual attention.
  • Submit gate is per-section, not per-grid. Only R&M is gated for now. Other sections (Gen & Admin, Energy, etc.) still submit as today. Easy to add more sections to the gate list later.
  • Capital exempt. Capital decisions happen on the Capital Forecast tab, not in the PM grid. Per-line PM gate here would duplicate work.
  • Bulk action requires reading the count + dollar exposure before confirming. The number "9 lines" and "$87,400" force the PM to acknowledge what they're sweeping.
  • Audit trail: every row review (typed or bulk) logged with user_id, timestamp, source (individual vs bulk), so the FA can see in the diff strip exactly how each line was decided.
  • Jump-to-next button keeps the PM moving without hunting for red rows.
