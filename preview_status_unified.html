3 · What changed

Before

  • Amber meant 8 different things; green meant opposites 30px apart
  • Green tile told you nothing about whether the wizard ran
  • Dashboard and wizard checked SharePoint differently → contradictory colors
  • Clicking a dashboard tile dropped you on a generic wizard page
  • "Nothing commits until Step 5" — false; and stale caches manufactured false alarms
  • Status shown twice per row; Maintenance Proof invisible

After

  • Amber = in SharePoint (with date) · Green = in a built budget · Red = missing/failed · Gray = setup
  • All-green row = the wizard ran. All-amber row = build me. Green+one-red = partial build
  • One server-side status model feeds both pages — they cannot disagree
  • Tile click lands on the exact wizard card, highlighted, same words
  • Auto-refresh on view + honest copy: files load on arrival, Step 4 generates
  • One Stage column · all six files visible · newer-file-after-build gets a marker
