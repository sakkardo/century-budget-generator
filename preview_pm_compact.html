What changed vs the live build

  • 14 columns → 7 columns by default. Prior, YTD, Accrual, Unpaid, Estimate, Forecast, $ Variance, % Change are hidden behind the ▶ row expander. Power users see them on demand; PMs aren't drowning in them.
  • 9 visual treatments → 3. Gray text for reference, blue input for action, red/green pills for delta. Killed the fx stickers (FA-only feature), the green borders on Estimate/Forecast cells (not editable here), the yellow Notes border (replaced with a tiny red dot when notes are needed).
  • "No change" button → inline icon button. Sits to the right of the Proposed input, same height as the input. Row heights stop jittering. Click the "= " icon to flat-line proposed = current.
  • Subtotal rows render derived pills. No more visual gap where data rows have pills and subtotals have blank cells. Now you see the rollup delta per category, which is useful.
  • Column order matches the decision flow. Identity → Current Budget → Proposed (the action) → Delta pills → Notes. Diagnostic comparisons (vs Forecast) hide in the detail panel where they belong.
  • Notes column gets full width. Small red dot on the left signals "this row needs a note" when delta > 10% and notes is empty.
