# Yardi Automation Patterns

## Architecture Overview

### Yardi Voyager Basics
- **Type:** ASP.NET WebForms application
- **URL:** `https://www.yardiasp13.com/03578cms/`
- **Century Management Instance:** 03578cms
- **Authentication:** Session-based (cookies)
- **Layout:** Heavy use of iframes/frames — main content loads inside nested frames

### Critical Limitation: Hidden Form Fields at Runtime
Yardi populates hidden form fields with JavaScript in the browser **after the page loads**. Raw HTML won't show the actual values:
- Static HTML shows `<input name="ReportMonitor" value="" />`
- Browser JavaScript fills it with the real value: `'../pages/SysConductorReportMonitor.aspx'`
- This is why browser automation (Puppeteer, Playwright) often fails — it tries to read or submit forms before JavaScript has run
- **Solution:** Use browser Console fetch scripts that inherit the session and already-populated form state

## Automation Pattern: Console Fetch Scripts

### Why Console Scripts Beat Browser Automation

| Method | Reliability | Complexity | Speed |
|--------|-------------|-----------|-------|
| Browser automation (Puppeteer, Playwright, Claude in Chrome clicking) | Low — frames break selectors | High | Slow |
| **Console fetch scripts** | **High — inherit session + form state** | **Low** | **Fast** |

### The Proven Pattern

1. **Capture one cURL** from DevTools Network tab (see a working form submission)
2. **Convert to a looping fetch script** (add error handling, polling, retry logic)
3. **Run in browser Console** while logged into Yardi
   - Console automatically inherits session cookies
   - No auth handling needed
   - Form fields are already populated by browser JS

### Example: Basic Report Submission via Console

```javascript
// Step 1: Submit the report request
const formData = new URLSearchParams();
formData.append('BPOSTED', '-1');  // CRITICAL: -1 = submission, 0 = page load (ignored)
formData.append('ReportMonitor', '../pages/SysConductorReportMonitor.aspx');
formData.append('Records', '');
formData.append('RptOutput', 'Filexlsx');
formData.append('__VIEWSTATE', document.querySelector('[name="__VIEWSTATE"]').value);
// ... add other hidden fields as needed

const submitResp = await fetch('https://www.yardiasp13.com/03578cms/pages/SysSqlScript.aspx?action=Filter&select=reports/...', {
  method: 'POST',
  body: formData,
  headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
  credentials: 'include'  // Include session cookies
});

const submitHtml = await submitResp.text();

// Step 2: Extract the new record ID from the response
const doc = new DOMParser().parseFromString(submitHtml, 'text/html');
const recordsField = doc.querySelector('input[name="Records"]');
const allRecords = decodeURIComponent(recordsField.value).split(',');
const newRecordId = allRecords[0];  // Newest ID is always first

console.log('Submitted report, record ID:', newRecordId);
```

## Critical Form Fields for Report Submission

### `BPOSTED` (Hidden Field) — The #1 Gotcha

| Value | Meaning | Server Action |
|-------|---------|---------------|
| `'-1'` | **Real form submission** | Queues the report, processes data |
| `'0'` | Page load / GET request | Ignores the data, just renders the page |

**Common Mistake:** Submitting with `BPOSTED=0` — the server accepts the POST but doesn't actually queue the report.

### `ReportMonitor` (Hidden Field) — Path to Polling Endpoint

- **Raw HTML value:** Empty string `""`
- **Browser JavaScript populates it with:** `'../pages/SysConductorReportMonitor.aspx'`
- **Why it matters:** The server uses this field to know where to send status updates. If empty, the server won't queue the report.
- **In your console script:** Hardcode it as `'../pages/SysConductorReportMonitor.aspx'`

### `Records` (Hidden Field) — Report Request IDs

- **Initial submission:** Send as empty string `''`
- **Server response:** Contains a new `<input name="Records">` with a comma-separated list of record IDs
- **Format:** IDs are URL-encoded with `%2C` for commas
- **How to parse:**
  ```javascript
  const allRecords = decodeURIComponent(recordsField.value).split(',');
  const newRecordId = allRecords[0];  // Newest is always first
  ```
- **Use case:** If you need to track the request or poll for completion, store `newRecordId`

### `RptOutput` (Hidden Field) — Output Format

- `'Filexlsx'` → Excel file (most common)
- `'FilePdf'` → PDF
- Other values exist but Excel is the standard for budget data

### `__VIEWSTATE` and Other ASP.NET Hidden Fields

- These are automatically generated by ASP.NET and validated on the server
- **How to get them:** `document.querySelector('[name="__VIEWSTATE"]').value`
- **Required:** Yes, always include them. Server will reject the POST if they're missing or invalid.
- **How to find all hidden fields in your form:** Use the DevTools Network tab to see the cURL, then extract all hidden field names and their current values from the page DOM

## Report Async Flow: From Submission to Download

### Step 1: POST Form Submission

```javascript
const formData = new URLSearchParams();
formData.append('BPOSTED', '-1');
formData.append('ReportMonitor', '../pages/SysConductorReportMonitor.aspx');
formData.append('Records', '');
formData.append('RptOutput', 'Filexlsx');
formData.append('__VIEWSTATE', document.querySelector('[name="__VIEWSTATE"]').value);

const submitResp = await fetch('https://www.yardiasp13.com/03578cms/pages/SysSqlScript.aspx?action=Filter&select=...', {
  method: 'POST',
  body: formData,
  credentials: 'include'
});

const submitHtml = await submitResp.text();
```

**Server Response:** HTML containing a new `Records` field with the submitted report's ID

### Step 2: Extract Record ID

```javascript
const doc = new DOMParser().parseFromString(submitHtml, 'text/html');
const recordsField = doc.querySelector('input[name="Records"]');
const allRecords = decodeURIComponent(recordsField.value).split(',');
const newRecordId = allRecords[0];
```

**Why:** The server doesn't immediately generate the report. It queues it and assigns an ID. You'll poll this ID to check for completion.

### Step 3: Poll Monitor Page (Every 3 Seconds)

```javascript
const MONITOR_URL = 'https://www.yardiasp13.com/03578cms/pages/SysConductorReportMonitor.aspx';

let downloadLink = null;
let attempts = 0;
const maxAttempts = 60;  // 3 seconds × 60 = 3 minutes timeout

while (!downloadLink && attempts < maxAttempts) {
  await new Promise(resolve => setTimeout(resolve, 3000));  // Wait 3 seconds

  const monitorResp = await fetch(MONITOR_URL, { credentials: 'include' });
  const monitorText = await monitorResp.text();

  if (monitorText.includes('sFileName')) {
    // Download link has appeared — extract it
    const match = monitorText.match(/sFileName[^"]*"([^"]+)"/);
    if (match) {
      downloadLink = match[1];
      console.log('Report ready. Download link:', downloadLink);
    }
  }

  attempts++;
}

if (!downloadLink) {
  console.error('Report generation timed out after 3 minutes');
}
```

**Key Points:**
- Poll **every 3 seconds** — polling too fast wastes bandwidth; polling too slow makes the UX feel slow
- Look for the string `'sFileName'` in the response HTML — this indicates the report file is ready
- Extract the download link with a regex: `/sFileName[^"]*"([^"]+)"/`
- **No record ID parameter needed** for the monitor URL — Yardi tracks it server-side based on the session

### Step 4: Download and Trigger Browser Download

```javascript
const downloadResp = await fetch(
  'https://www.yardiasp13.com/03578cms/SysShuttleDisplayHandler.ashx?file=' + encodeURIComponent(downloadLink),
  { credentials: 'include' }
);

const blob = await downloadResp.blob();
const url = URL.createObjectURL(blob);
const a = document.createElement('a');
a.href = url;
a.download = 'Report.xlsx';  // Filename for the browser download
a.click();
URL.revokeObjectURL(url);

console.log('Download triggered');
```

**Handler:** `SysShuttleDisplayHandler.ashx` is Yardi's file download handler. Pass the file path as a query parameter.

## Record ID Extraction Deep Dive

### Why It Matters

After you POST a report request, the server returns HTML with a **new** `Records` field. This field contains a comma-separated list of report IDs (all reports submitted in this session).

### Parsing Pattern

```javascript
// 1. Parse the HTML response
const doc = new DOMParser().parseFromString(html, 'text/html');

// 2. Find the Records input
const recordsField = doc.querySelector('input[name="Records"]');

// 3. URL-decode the value (Yardi encodes commas as %2C)
const decoded = decodeURIComponent(recordsField.value);
// Example: "12345%2C12344%2C12343" → "12345,12344,12343"

// 4. Split by comma to get individual IDs
const allRecords = decoded.split(',');

// 5. The newest ID is always first
const newRecordId = allRecords[0];
```

### Why "Newest is First"

Yardi appends new IDs to the front of the list. If you submit three reports:
1. First submission: `Records = "100"`
2. Second submission: `Records = "101,100"`
3. Third submission: `Records = "102,101,100"`

So when you extract `allRecords[0]`, you always get the most recent submission.

## Polling Pattern (General)

Use this pattern whenever you need to wait for async server-side work:

```javascript
async function pollUntilReady(checkFn, maxAttempts = 60, intervalMs = 3000) {
  for (let attempt = 0; attempt < maxAttempts; attempt++) {
    const result = await checkFn();
    if (result) {
      return result;
    }
    await new Promise(resolve => setTimeout(resolve, intervalMs));
  }
  throw new Error('Polling timed out');
}

// Usage:
const downloadLink = await pollUntilReady(async () => {
  const resp = await fetch(MONITOR_URL, { credentials: 'include' });
  const text = await resp.text();
  const match = text.match(/sFileName[^"]*"([^"]+)"/);
  return match ? match[1] : null;
});
```

## Common Pitfalls

### 1. Using `BPOSTED=0` Instead of `BPOSTED='-1'`

**Pitfall:** The POST succeeds (HTTP 200), but the report never queues. You poll the monitor page forever and get nothing.

**Root cause:** `BPOSTED=0` signals a page load, not a form submission. The server accepts the request but ignores the form data.

**Fix:** Always use `BPOSTED='-1'` for actual submissions.

### 2. Leaving `ReportMonitor` Empty

**Pitfall:** The report doesn't queue. The monitor page never shows a download link.

**Root cause:** The server needs to know where to send progress updates. An empty `ReportMonitor` field breaks the queue.

**Fix:** Hardcode `ReportMonitor='../pages/SysConductorReportMonitor.aspx'`. The server fills it in browser-side JavaScript, but your fetch script must provide it manually.

### 3. Trying Browser Automation (Puppeteer, Playwright, etc.)

**Pitfall:** Selectors fail to find elements. Frame switching is inconsistent. Form fields aren't populated because JavaScript hasn't run yet.

**Root cause:** Yardi's iframe-heavy architecture doesn't work well with traditional browser automation. Hidden form fields are populated by JavaScript at runtime, not in static HTML.

**Fix:** Use console fetch scripts instead. They inherit the session and form state without needing to click through the UI.

### 4. Entity Codes with Leading Zeros or String Format

**Pitfall:** Report filtering fails or returns wrong data.

**Root cause:** Entity codes are numeric in the database. Passing `"0148"` (string) might not match the code `148` (number).

**Fix:** Extract entity codes as numbers, pass them as integers in the fetch request. Example: `entityCode: 148` (not `"0148"` or `"148"`).

### 5. Not Including Required Hidden Fields

**Pitfall:** Server returns `400 Bad Request` or silently ignores the form.

**Root cause:** ASP.NET validates hidden fields like `__VIEWSTATE` and `__EVENTVALIDATION`. If they're missing or stale, the server rejects the request.

**Fix:** Before writing your fetch script, capture a real form submission with DevTools Network tab. Extract **all** hidden field names. In your script, use `document.querySelector('[name="FIELDNAME"]').value` to get current values from the page.

### 6. Polling Too Aggressively

**Pitfall:** Yardi server returns 429 (Too Many Requests) or becomes unresponsive.

**Root cause:** Polling every 100ms hammers the server.

**Fix:** Poll every 3 seconds or use exponential backoff. Yardi reports typically take 5-30 seconds to generate.

### 7. Not Using `credentials: 'include'` in Fetch

**Pitfall:** Session cookies aren't sent. The server returns 401 (Unauthorized) or redirects to login.

**Root cause:** Fetch doesn't send credentials by default (security measure). Yardi requires session cookies.

**Fix:** Always include `credentials: 'include'` in fetch options.

## Session and Authentication

### How Sessions Work

- Yardi uses cookie-based session authentication
- When you log in via the web UI, the browser stores a session cookie
- Browser Console scripts automatically inherit these cookies via `credentials: 'include'`

### Running Console Scripts

1. Open DevTools (F12) in a Yardi page
2. Go to Console tab
3. Paste your fetch script (can be multi-line in the Console)
4. Press Enter
5. Wait for the script to complete — it will log progress and the final result

### No Manual Auth Needed

- You do **not** need to hardcode usernames or passwords
- You do **not** need to add Authentication headers
- Just ensure the console script runs in the context of an already-logged-in Yardi page

## File Paths and URLs

### Base URL
```
https://www.yardiasp13.com/03578cms/
```

### Common Endpoints
- **Report submission:** `.../pages/SysSqlScript.aspx?action=Filter&select=reports/...`
- **Monitor/polling:** `.../pages/SysConductorReportMonitor.aspx`
- **File download:** `.../SysShuttleDisplayHandler.ashx?file=...`

### Report Selection Strings

The `select=` parameter in the URL specifies which report to run. Examples:
- YSL Annual Budget: `reports/rs_YSL_CMS_Annual_Budget_60612`
- Other reports follow the pattern: `reports/[REPORT_CODE]`

To find a report code:
1. Navigate to the report in the Yardi UI
2. Look at the URL in the address bar — the `select=` parameter shows the code
3. Use that code in your fetch script

## Debugging Tips

### 1. Log All Responses

```javascript
const resp = await fetch(url, options);
const text = await resp.text();
console.log('Response status:', resp.status);
console.log('Response body (first 500 chars):', text.substring(0, 500));
```

### 2. Check for Hidden Fields

```javascript
// See what hidden fields are on the current page
const hiddenFields = document.querySelectorAll('input[type="hidden"]');
hiddenFields.forEach(field => {
  console.log(field.name, '=', field.value);
});
```

### 3. Verify Session is Active

```javascript
// Navigate to any Yardi page. If the page loads normally (not a login redirect), session is active.
const sessionCheck = await fetch('https://www.yardiasp13.com/03578cms/', { credentials: 'include' });
console.log('Session active:', sessionCheck.ok);
```

### 4. Network Tab in DevTools

- Open DevTools → Network tab
- Perform the action manually in the Yardi UI
- Right-click on the request → Copy as cURL
- Paste the cURL into an online tool like `curlconverter.com` to see the equivalent JavaScript fetch
- Use that as a template for your console script
