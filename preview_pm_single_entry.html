PM Dashboard - Single Entry Model (Full Column Set)

Same PM dashboard you have today, all 13 columns intact. The only change: the PM types the 2027 Proposed Budget directly, and the Increase % / Increase $ columns become read-only and derive from (Proposed - Current Budget). All reference columns (Prior, YTD, Estimate, Forecast, Curr Budget) and diagnostic columns ($ Variance vs Forecast, % Change vs Forecast) stay exactly as they are.

What's different from today: The Increase % and Increase $ inputs become derived pills. The 2027 Proposed cell becomes the only editable number in the row (plus Notes). Every other column - Prior Year, YTD, Estimate, Forecast, Curr Budget, $ Variance, % Change - stays identical to the current build.

PM picks Increase % or Increase $, Proposed is computed.

2 editable cells. PM has to decide which one to use.

After (proposed)

PM types into Proposed, deltas auto-fill.

1 editable cell. Deltas live-update.
