The structural move — two pages, not one

Your instinct is right. The mental model is:

  • "What needs my attention?" — health, readiness, warnings, action items. A landing page.
  • "Let me edit the numbers." — the workbook tabs. A focused page.

Two separate URLs:

  • /dashboard/<ec> → Action Center landing (NEW)
  • /dashboard/<ec>/workbook → Workbook view (existing, stripped down)

The Action Center is the default landing. Click "Open Workbook" or any tab name to switch.

Click-through behavior — fixing the broken jumps

Every action button on the Action Center goes somewhere useful. Three patterns:

Pattern 1: Inline expand (most readiness items)

Click "Review" on "6 approved-file labels won't aggregate" → the item EXPANDS to show the full list inline (see Action Center mockup above). No page navigation. No JSON. Just the data, right there.

Pattern 2: In-page anchor + tab switch (most warnings)

Click "+ Add Row" on "1 unmapped GL: 4520-0000" → switches to the workbook view + jumps to Summary tab + scrolls to the orphan warning row + opens the Add Row modal pre-filled.

Pattern 3: Dedicated review page (a few cases)

Click "View" on "FY2025 audit confirmed" → navigates to /audited-financials/review/<upload_id> (already exists). Click "Open canonical map" → admin page at /admin/portfolio-health.

What's gone

  • Jumping to raw JSON endpoints. /api/wizard/<ec>/scan-findings is for the wizard sidebar JS only. The user never sees it directly.
  • Anchor links to non-existent IDs. #tab=Payroll would silently do nothing if the JS handler isn't wired. Now every link is verified to have a receiver.
  • Action buttons that say "Generate" before all upstream gates are green. Now disabled with a tooltip.

Implementation plan

Routing

  • /dashboard/<ec> → renders the new Action Center (currently renders the workbook page)
  • /dashboard/<ec>/workbook → renders the workbook view (the existing template, stripped of readiness/sidebar/warnings)
  • The two templates share the header strip, pipeline, KPI cards. Different bodies.

Total: ~10-12 hours of work, one commit. Risk is medium — touches the most-visited page. Mitigation: roll out behind a query param flag (?action_center=1) for a day, then flip default.

Rollout safety

  • Ship behind ?action_center=1 first — FA visits /dashboard/148?action_center=1 to see the new view. Default URL still renders existing dashboard.
  • Verify it works on 168 + 148 + a condo + a building with no data
  • Flip the default after FA validates
  • Old workbook view always reachable via ?action_center=0 for a week as fallback
  • Keep the existing IDs/JS populators intact (same defensive pattern as Phase 2)
