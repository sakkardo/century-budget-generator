Why this matters

  • Same visual language for every status signal — dots + label + detail + action button. Today the audit uses 4-step dots, the period uses a 13px sentence, the diff uses pill-tagged bullets, and readiness uses a different dot system.
  • One mental model: top section is "what changed since I left", bottom section is "what's still left to do". Today the FA has to mentally separate these from format alone.
  • Collapsible as a unit — when all gates green and no recent activity, collapses to a single line. Today four bars stay visible.
  • Implementation: combine renderAuditStatusBanner + renderPeriodBanner + renderDiffStrip + renderReadinessInspector into one renderBuildingStatus(data). Existing endpoints unchanged.

The fix

Extract a single _BASE_STYLES Python string with the CSS vars (--blue, --green, --gray-100, etc.) and prepend it to every render_template_string call. The RE Tax tab's hardcoded inline styles get replaced with var(--*) references. Visual cohesion in 30 minutes — no logic change, no math change.

Preview 3 — Signed cookie code preview

Today — unsigned, trivially forgeable

# workflow.py:4650
def _read_fa_id_from_cookie():
    try:
        raw = request.cookies.get("century_fa_id")
        if not raw: return None
        uid = int(raw)
        user = User.query.filter_by(id=uid).first()
        if not user: return None
        return uid
    except Exception:
        return None

# Anyone can do:
# document.cookie = "century_fa_id=18"
# and become Jennifer Murman.

Signed — forgery-proof

# workflow.py — at module top
from itsdangerous import URLSafeSerializer, BadSignature

def _fa_signer():
    return URLSafeSerializer(
        current_app.config["SECRET_KEY"],
        salt="century-fa-id",
    )

def _read_fa_id_from_cookie():
    raw = request.cookies.get("century_fa_id")
    if not raw: return None
    try:
        uid = _fa_signer().loads(raw)
        return int(uid)
    except (BadSignature, ValueError, TypeError):
        return None

Plus the matching change in api_set_whoami to encode with signer.dumps(user.id) when setting the cookie. Forgery requires SECRET_KEY — which is in Railway env, not in any client.

You're not on fire. You have one big architectural problem (the two monoliths) and three medium ones (no tests, hardcoded entity defaults, SharePoint filename matching). The product itself is well-conceived and the recent commits (auto-save fixes, RE Tax overhaul, FA identity, diff strip) are solid. Everything in this review is polish.

The 5 quick wins + Day 2 UX changes are the highest-ROI work for this week. The deeper refactors (workflow.py split, JS extraction) wait until rollout teaches you what the natural seams are.
